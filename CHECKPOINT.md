## 📱 Проект

**MoodOS** — мобильное приложение эмоциональной регуляции на **Capacitor/Ionic**
- Репозиторий: `https://github.com/serjoleinic2-cloud/moodos-app`
- Файлы приложения лежат в папке `www/`
- Точка входа: `www/index.html` + `www/js/app.js`
- Стили: `www/css/style.css`

## 🎨 Философия и дизайн

**Стиль: неоморфизм** — светлый фон `#e0e5ec`, тени `#b8bec7` / `#ffffff`
```css
/* Выпуклый элемент */
box-shadow: 6px 6px 14px #b8bec7, -6px -6px 14px #ffffff;
/* Вдавленный (активный) */
box-shadow: inset 4px 4px 8px #b8bec7, inset -4px -4px 8px #ffffff;
```

**Философия приложения:**
- MoodOS — **попутчик**, не советчик
- Не даёт советов которые юзер не просил
- Объясняет состояние (причины), а не оценивает
- «Я с тобой уже 7 дней» — дружеский тон, не клинический
- Всё на **русском языке** (UI полностью русифицирован)
- Поддержка 4 языков: RU, EN, ES, UK (через offline-ai.js и планируемый i18n.js)

## ✅ Что сделано и работает

### Онбординг (onboarding.js)
- 6 шагов: приветствие → состояние → лекарства → эффект → напоминалка → калибровка
- Шаги 4-5 пропускаются если юзер не принимает лекарства
- Прогресс-точки, кнопки Назад/Далее, счётчик «Шаг X из Y»
- Показывается ТОЛЬКО при первом запуске (флаг `onboarding_done` в localStorage)
- **НЕ показывается повторно** — это критично

### Профиль юзера (user-profile.js) — путь: `www/js/services/`
- `getProfile()` / `saveProfile()` — хранит baseFeeling, takesMeds, medEffect, medReminder, moodBaseline
- `getMoodBaseline()` — возвращает откалиброванный базовый уровень (учитывает лекарства)
- `getMedContext()` — строка-подсказка для insight-engine
- `isOnboardingDone()` / `markOnboardingDone()`
- `shouldShowMonthlyCheck()` / `markMonthlyCheckDone()` — раз в 30 дней

### Ежемесячный чек (monthly-check.js)
- Всплывает снизу через 2.5 сек после запуска (только если принимает лекарства)
- Варианты: «Да всё так же» / «Кое-что изменилось» / «Больше не принимаю» / «Напомни позже»

### Движки аналитики (services/)
- **pattern-engine.js** — лучший час дня, лучший день недели, триггеры из заметок (60+ паттернов, RU/EN/ES/UK)
- **resilience-engine.js** — индекс устойчивости 0-100, скорость восстановления, тренд
- **insight-engine.js** — explainCurrentState, getTodayForecast, getAmnesiaReminder, getMicroHabit

### Экран Insight (screens/insight.js)
- Кольцо устойчивости (Canvas)
- «Почему так себя чувствую» — живые фразы
- «Прогноз на сегодня»
- «Ты уже справлялся» — блок амнезии
- «Скрытые триггеры» — с иконками
- «Что поможет сейчас» — микро-привычка

### Экран Report (screens/report.js)
- Переключатель 7 дней / 30 дней / Всё время
- Карточки 2×2: среднее настроение, стабильность, записей, практик
- Линейный график (Chart.js)
- Лучший и сложный момент
- Общий вывод — умный текст

### Экран Stability (screens/stability.js)
- Кольцевой прогресс-бар (Canvas, без Chart.js)
- Карточки: волатильность, среднее за 14 дней, тренд, кол-во записей
- Линейный график за 14 дней
- Прогресс-бары для последних 10 записей

### История (screens/history.js)
- Таймлайн: настроение + заметки + голос + сессии + фото
- Фильтр по дате
- Кнопка 📷 — добавить фото (сжимает до 800px, JPEG 80%)
- Мусорка 🗑 на каждой карточке + диалог «Удалить запись?»
- Детальный просмотр каждой записи

### Настройки (screens/settings.js)
- Раздел «Здоровье и самочувствие»: приём лекарств, как влияет, напоминание
- Базовое состояние — слайдер
- Модалки снизу для редактирования каждого поля
- Сюда юзер попадает если захочет изменить данные о лекарствах

### Offline AI (ai/offline-ai.js)
- 4 языка: RU, EN, ES, UK
- Автоопределение языка через `navigator.language` или `localStorage.app_language`
- 3 пула ответов: neutral / low / high — по уровню настроения
- Умный подбор по ключевым словам в тексте (сон, усталость, работа, тревога и т.д.)
- ~15 ответов на язык × 3 состояния = ~180 вариантов

### Home (screens/home.js)
- Дата последнего сохранения: «сегодня в 14:32» / «вчера» / «3 мар»

## 💡 Подсказки для нового Claude

- Всегда читай файл с GitHub перед правкой (`web_fetch` на raw ссылку)
- Не используй `?.property =` для присваивания — это синтаксическая ошибка
- При создании файлов сначала проверяй синтаксис через bash: `node --input-type=module < file.js`
- Все новые экраны рендерят HTML в контейнер `data-screen="screenname"` через `innerHTML`
- Chart.js подключён глобально через CDN в index.html — доступен как `window.Chart`
- Canvas графики уничтожай перед пересозданием: `Chart.getChart(canvas)?.destroy()`
- localStorage ключи: `mood`, `startDate`, `mood_history`, `notes_history`, `voice_history`, `session_history`, `photo_history`, `user_profile`, `onboarding_done`, `med_reminder`, `med_monthly_check`, `app_language`
